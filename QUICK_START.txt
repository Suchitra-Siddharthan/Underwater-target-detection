HOW TO USE
═════════════════════════════════════════════════════════════════

STEP 1: LOGIN
  → Navigate to login page
  → Create account or login with existing credentials

STEP 2: UPLOAD & DETECT (Original Feature)
  → Click "Detection" in header
  → Upload an image
  → Click "Detect Targets"
  → See results with annotated image

STEP 3: CHECK ANALYTICS (NEW)
  → Click "Analytics" in header
  → See statistics about all your detections
  → Refresh to update numbers
  → View breakdown by creature class

STEP 4: EXPERIMENT WITH THRESHOLDS (NEW)
  → Click "Threshold" in header
  → Upload an image
  → Adjust threshold slider from 0.00 to 1.0
  → See how detection results change
  → Compare different threshold settings

  Threshold Meanings:
    0.0  = Show all detections (many false positives possible)
    0.25 = Balanced (default)
    0.5  = More selective
    0.7  = Very strict (only high confidence)
    1.0  = Maximum filtering (may miss objects)

STEP 5: REVIEW HISTORY
  → Click "History" in header
  → See all past detections with images
  → Download individual results


API USAGE
═════════════════════════════════════════════════════════════════

Get Analytics:
  curl -H "Authorization: Bearer {TOKEN}" \
    http://localhost:8000/features/analytics/summary

Run Threshold Prediction:
  curl -X POST \
    -H "Authorization: Bearer {TOKEN}" \
    -F "file=@image.jpg" \
    -F "confidence_threshold=0.5" \
    http://localhost:8000/features/predict_with_threshold

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════

Backend not responding?
  → Check: python -m uvicorn main:app --reload
  → Visit: http://localhost:8000/health

Frontend not showing new features?
  → Check: npm run dev is running
  → Browser cache: Clear or hard refresh (Ctrl+F5)
  → Logged in? Features only visible when authenticated

Analytics shows no data?
  → Normal if no detections uploaded yet
  → Upload and detect images first
  → Then view analytics (data aggregates)

Threshold prediction fails?
  → Check image file format (jpg/png)
  → Check threshold is between 0.0 and 1.0
  → Check backend is running
